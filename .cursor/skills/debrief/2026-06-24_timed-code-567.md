## Conclusions

### Final approach (pass)

- `freq` map for `s1`; `while` scan of `s2` with `le` = window start
- Extend while char in map with remaining count; `count` ‚Üí `len(s1)` ‚Üí return `True`
- On mismatch while `in_window`: reset map, `i = le + 1`, `continue`
- On success path: `i += 1`

### What went well

- Opened with hash-map + two-mode narrative (interview-safe)
- Caught `hash["c"]` literal bug early
- Fixed operator precedence on reset condition
- **`elif` ‚Üí `if`** so current char re-processed after reset
- **`for` ‚Üí `while`** so `i = le + 1` actually controls flow
- Renamed `hash` ‚Üí `freq`; count-up invariant clear

### Bugs fixed along the way

| Bug | Fix |
|-----|-----|
| `hash["c"]` literal | `freq[c]` |
| `or (c not in hash2)` without `substr` guard | `if substr and (...)` |
| Reset skips current char | separate `if` after reset block |
| `i = le + 1` in `for` loop | `while` + manual `i` |
| Missing `i += 1` | increment after successful consume |
| Overlap cases (`adc`/`dcda`, `ab`/`aab`) | `le` + jump to `le+1` |
